## 소개

NotiToss는 같은 Wi-Fi 네트워크 내에서 안드로이드 기기 간 알림을 전달하는 앱입니다.
클라우드 서버 없이 기기 간 직접 통신하므로 빠르고 안전합니다.

**주요 사용 사례:**
- 메인폰 알림을 태블릿에서 확인
- 업무폰 알림을 개인폰으로 전달
- 한 기기의 알림을 여러 서브 기기에서 동시 수신

## 주요 기능

### 알림 미러링
- **1:N 멀티 디바이스** — 송신 기기 1대 → 수신 기기 여러 대 동시 전달
- **앱별 선택** — 원하는 앱의 알림만 골라서 전달
- **아이콘 전달** — 알림 아이콘과 앱 아이콘을 함께 전달
- **전화 알림** — 수신 시 헤드업 알림 + 진동, 통화 종료 시 자동 해제

### 페어링
- **간편 코드 인증** — 6자리 코드로 안전하게 페어링
- **기기별 관리** — 개별 추가/해제, 연결 상태 실시간 확인
- **양방향 해제** — 송신/수신 어느 쪽에서든 해제 가능

### 기타
- **수신 로그** — 수신된 알림 기록 (앱 재시작 후에도 유지)
- **홈 위젯** — 최근 알림을 위젯으로 빠르게 확인
- **방해금지 무시** — 전화 알림은 DND 모드에서도 전달
- **로그 건수 설정** — 20~100건 사이 조절 가능

## 동작 방식

```
┌──────────────┐        Wi-Fi (HTTP)        ┌──────────────┐
│   송신 기기    │  ───────────────────────▶  │   수신 기기    │
│  (메인폰)     │                            │ (태블릿/서브폰) │
│              │   알림 감지 → 즉시 전달       │              │
│  Notification │                            │  알림 표시 +   │
│   Listener   │                            │  로그 저장     │
└──────────────┘                            └──────────────┘
```

- 클라우드 서버 없이 **로컬 네트워크 직접 통신** (HTTP, 포트 8080)
- **EncryptedSharedPreferences**로 토큰 암호화 저장
- 알림 아이콘은 WEBP 압축으로 전송하여 **배터리/네트워크 절약**

## 기술 스택

| 영역 | 기술 |
|------|------|
| UI | Jetpack Compose + Material 3 |
| HTTP 서버 | NanoHTTPD |
| HTTP 클라이언트 | OkHttp |
| 알림 캡처 | NotificationListenerService |
| 백그라운드 | WorkManager + Foreground Service |
| 데이터 저장 | EncryptedSharedPreferences (AES256-GCM) |
| 위젯 | Glance AppWidget |
| 언어 | Kotlin |

## 요구 사항

- Android 9.0 (API 28) 이상
- 두 기기가 같은 Wi-Fi 네트워크에 연결

## 사용 방법

1. 두 기기에 NotiToss를 설치합니다
2. **메인폰**에서 "송신", **서브 기기**에서 "수신" 역할을 선택합니다
3. 수신 기기에서 "페어링 시작" → 6자리 코드 표시
4. 송신 기기에서 수신 기기의 IP와 코드를 입력합니다
5. 모니터링할 앱을 선택하면 알림 전달이 시작됩니다
